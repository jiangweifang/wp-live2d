V1 模型的两种使用方式

V1 模型（基于 journey-ad / Cubism 2~3 系列）始终免费，常见模型包括 22 娘 / 33 娘 / Pio / Tia / Shizuku / 舰これ Murakumo / 超次元海王星等。WordPress 后台 → Live2D 设置 → 「创意工坊 API 类型」前两个选项都是 V1。

方式 ①：本地（local）​

项目说明
数据来源插件 assets/v1/ 目录内置的 V1 模型
提供方式本站 REST 路径 /wp-json/live2d/v1/model/*
是否依赖外网否，完全在你自己的站点提供
适用场景默认推荐 —— 装上插件什么都不动就能用
限制模型固定为插件自带的几个；想用其他 V1 模型请走「远程创意工坊」

怎么启用：「创意工坊 API 类型」选 本地，保存设置即可。前台立刻就有看板娘了。

方式 ②：远程创意工坊（remote）​

项目说明
数据来源从 www.live2dweb.com 拉模型清单
工作方式拉到清单后下载到本地，再走 V1 加载链路
下载位置wp-content/plugins/live-2d/model/
是否依赖外网首次需要访问 www.live2dweb.com 拉清单 + 下载文件；下载完成后即可断网使用
适用场景想用比插件自带更多的 V1 皮肤（22 / 33 各种贴图变体、Pio 多套服装等）

怎么启用：「创意工坊 API 类型」选 远程创意工坊，保存。在前台的看板娘工具栏点「换模型」，会拉到完整的远程模型库列表。

两者怎么选？​

  • 第一次接触本插件 / 默认体验 → 选「本地」即可。
  • 想用更多种类的 V1 模型 → 选「远程创意工坊」，然后在前台点「换模型」浏览全部库存。

提示：两种模式都不需要付费、不需要登录。如果想用 Cubism 4+ MOC3 模型（V2），请走 V2 自定义模型。

V1 与浏览器扩展的关系​

浏览器扩展端有一个独立的「V1 模型库」标签页，逻辑与 WordPress 插件的「远程创意工坊」类似：列表 → 下载到本地（IndexedDB） → 选用。但两边的下载库存是分开维护的：扩展端的 V1 列表是扩展自己内置的 catalog（PNG 缩略图 + *.textures.json），不会向 www.live2dweb.com 请求。详见 Chrome / Edge 扩展。

常见误解​

  • "V1 是不是不能用了？" — 不是。V1 链路从 2.1.3 起反而完全本地化（不再依赖 fghrsh.net 等第三方源），稳定性比以前好得多。它只是不会再加 新功能（融合变形、超采样这些 Cubism 4+ 才有的特性）。
  • "自己搭一个 V1 模型服务器？" — 可以，参考 fghrsh/live2d_api（外部）。但从 2.1.3 起本插件已经内置常用 V1 模型，一般不再需要第三方 API。
