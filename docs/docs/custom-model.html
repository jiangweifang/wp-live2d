使用 V2 自定义模型（Cubism 4+）

本页只讲 V2（Cubism 4+ / 5）模型。如果你想用 22/33/Pio/Tia 这种老模型，那是 V1，请走插件后台「创意工坊 API 类型 = 本地 / 远程创意工坊」，不需要付费，也不需要本页的任何步骤。先去看 V1 / V2 是什么。

1. 先付费

V2 模型走的是 Live2D Cubism 4+ Web SDK 加载器。Live2D 株式会社要求第三方扩展性应用使用 Cubism 4+ 时签订商业合约，本插件作者已经签了长期合约并按其要求分发正版 Cubism Core，所以这条链路对站长收一次性授权费。

  1. 在 www.live2dweb.com 完成账号注册与付费。
  2. 在 WordPress 后台保持登录状态（这一步是为了拿到与你账号绑定的 token，让插件能向 live2dweb.com 验证授权）。

没付费 / 没登录的状态下，插件后台「创意工坊 API 类型」里 自定义 这一项会被禁用或提示需要登录。

2. 准备好你的 V2 模型文件

V2 模型是 Cubism 4+ 导出的，文件清单是 *.model3.json，主体是 *.moc3，再加上贴图、动作、表情等。

2.1 文件清单要求

  • 清单文件名一般是 {模型名}.model3.json。
  • 路径里 不能有中文。
  • 如果你的站点是 https，那么清单 以及它引用到的所有文件（.moc3、贴图、motion3.json、physics3.json、cdi3.json 等）都必须是 https，否则浏览器会因为混合内容 / 跨域报错。
  • 跨域：把模型放到 OSS / COS / 自家 CDN 时，请正确开启 CORS（一般加上你的站点域名作为允许的 Origin 即可）。

2.2 把模型放在哪里？

排序按"推荐程度从高到低"：

  1. 对象存储 + 防盗链（首选，自带版权保护）：阿里云 OSS、腾讯云 COS、又拍、七牛等。
    • 开启 Referer 白名单，只允许你的站点访问。
    • 套一层自家 CDN 时记得继承防盗链规则。
  2. 插件目录下的 model 文件夹（最简单，但需要你自己控制流量）： wp-content/plugins/live-2d/model/{模型名}/{模型名}.model3.json 如果没有 model 文件夹，自己建一个。注意 WordPress 的 URL Rewrite 在某些主题 / 安全插件下可能会误拦截，访问不到时先确认这个 URL 浏览器能直接打开。
  3. 开启「缓存到本地」让插件接管：填 modelAPI 之后，在防盗链选项里选「缓存到本地」，插件会把对应文件全部下载到你站点的 wp-content/plugins/live-2d/model/，并对前端只暴露临时签名 URL，访客 F12 也看不到源站。详见防盗链页。
  4. 放到 GitHub Pages / jsDelivr 这种公开静态托管：能跑，但大陆地区不推荐，jsdelivr 实际还是会回源 raw.githubusercontent.com，速度不稳定，也无法防盗。

3. 在后台填入 modelAPI

WordPress 后台 → Live2D 设置 → 创意工坊 API 类型选「自定义」。下面有 modelAPI 输入框，有两种填写方式，由结尾字符决定走哪一种：

写法结尾作用下方 modelDir 列表
① 直链单模型以 .json 结尾直接加载该 model3.json，不再做任何路径拼接。忽略
② 目录根 + 多模型以 / 结尾把它当作根目录，配合下方 modelDir 列表，按 {modelAPI}{名称}/{名称}.model3.json 自动拼接成完整 URL。必须填，每行一个子目录名（必须与 {名称}.model3.json 完全一致）。

示例

① 直链：

https://cdn.example.com/live2d/haru/haru.model3.json

② 目录根 + 多模型：

modelAPI:  https://cdn.example.com/live2d/

modelDir:


    

  • haru
    • 

  • mao
    •
  • shizuku

实际加载时插件会拼成：

https://cdn.example.com/live2d/haru/haru.model3.json
https://cdn.example.com/live2d/mao/mao.model3.json
https://cdn.example.com/live2d/shizuku/shizuku.model3.json

前台「眨眼按钮」会在这几个模型之间切换。

注意：使用自定义模型后，前台 Live2D 的"上一个 / 下一个"切换按钮在很多模型组合下没意义，建议在后台关闭那两个按钮。

4. 模型贴图（材质）大小

请控制每张贴图不超过 2048×2048。Cubism 4+ 默认贴图就比较大，叠几张过大的 PNG 很容易把首屏加载时间拖到 5~10 秒以上。如果模型本身导出时贴图分辨率超标，可以在 Cubism Editor 里"嵌入纹理图集 → 重新分配尺寸"。

5. 进阶：超采样（仅 V2）

插件 2.1.3 起新增 超采样功能，可以让贴图在高分屏上更清晰。仅对 V2（model3.json）模型生效，V1 链路不会响应这个开关。

6. 安全 / 版权要点

  1. 请自行保证自己模型的版权。最佳做法：上传到带防盗链的对象存储 + 开启 CORS 限定来源。
  2. 创意工坊目前不支持自行上传 —— 因为我们没办法替每个上传者保护版权。
  3. 不要把 OSS 的 SecretKey 写进 WordPress 设置项。如果一定要走私有读 + 签名链接，请走 wp-config.php 环境变量，见防盗链页底部说明。

有问题欢迎在 GitHub Issues 中反馈。
