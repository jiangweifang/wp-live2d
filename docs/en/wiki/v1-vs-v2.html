The plugin maintains two completely independent Live2D loaders. If this is your first contact with the project, take two minutes to read this page before picking which path to use.

One-line summary

  • V1 models = the legacy loader based on journey-ad/live2d_src, targeting Cubism 2.x / 3.x models. Manifest file is usually model.json, the moc extension is .moc. Common models: 22-musume / 33-musume, Pio / Tia, Shizuku, Kantai-Collection Murakumo, Hyperdimension Neptunia, etc.
  • V2 models = Cubism 4 / 5 series (including any future versions). Manifest file is *.model3.json, the moc extension is .moc3. This is the format that Live2D Inc. is currently maintaining.

Naming note: older docs used to call Cubism 4+ models "v3 or higher". Starting with plugin 2.1 we standardized on the name V2. In this repository, "V2" does not mean Cubism 2 — it means the second generation of the plugin's loader. If you see "v3 model" or "V3 model" in old issues, please read it as V2.

How to tell whether your model is V1 or V2

Just check the manifest file:

Model typeManifest file name"version" field in the manifestMain moc file
V1 (legacy)model.jsonnone / any*.moc
V2 (current)*.model3.jsonstarts at "Version": 3 (this is the Cubism Web SDK number, not the model spec number)*.moc3

The frontend picks the V2 loader when the model spec version is ≥ 3, otherwise it falls through to V1.

Why is V2 paid?

Because Live2D Inc. requires third-party extensible applications to sign a commercial agreement before using the Cubism 4+ Core. The plugin author has signed a long-term contract and ships the official Cubism Core under that contract, so the V2 path bills site owners a one-off licensing fee — basically passing the upstream license cost on to the sites that actually use V2. V1 models do not depend on the Cubism Core, so they remain free forever.

Which one should I pick?

  • I just want a moving 2D character keeping me company on my blog → install the plugin, keep the defaults; you are already running the bundled V1 models (Local).
  • I want one of the popular ready-made skins (22-musume, Pio, …) → still V1, use either "Local" or "Remote workshop".
  • I have a MOC3 model I made or bought myself → use the V2 custom URL: pay first, then follow the V2 custom model guide.
  • I want maximum hotlink protection for my V2 model → on top of the custom URL also enable "cache locally"; see the hotlink page.

Common misunderstandings

  • "Is V1 deprecated?" No. Since 2.1.3 the V1 loader is in fact fully local (no more third-party sources such as fghrsh.net), so it is much more stable than before. It just will not gain new features like blend shapes or super-sampling that are Cubism 4+ exclusives.
  • "Is V2 the same thing as Cubism 2?" No. V2 in this repository is the second generation of the loader, which targets Cubism Web SDK 4 / 5. Cubism 2 is the ancient version and lives under the V1 loader.
  • "The old docs talk about v3 models…" The "v3" in the old docs is the same thing as today's V2. Treat any "v3 / V3" wording as V2.
