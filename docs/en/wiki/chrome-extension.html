Besides the WordPress plugin, this project also ships a Chrome / Edge browser extension called Live2D Web Canvas that pins a Live2D character to the bottom-right of any webpage. The extension and the WordPress plugin are two independent products; they share the same login account but their settings are completely separate.

1. What can it do?

  • Pin a Live2D character to the bottom-right corner of every site you visit (it does not read anything from the host page; it only animates the character based on the live mouse position — see the privacy policy).
  • Ships with several classic V1 models (22-musume / 33-musume, Pio / Tia, Shizuku, Kantai-Collection Murakumo, Hyperdimension Neptunia, …) downloadable into IndexedDB for offline use.
  • Once you sign in to a paid account, you can plug in your own modelAPI to load V2 (Cubism 4+) models.
  • Adjustable head anchor, scale, character position, canvas size, tip-bubble frequency, toolbar buttons, etc.

2. Installation

Search for Live2D Web Canvas on the Chrome Web Store / Edge add-ons store and install it. After installation an extension icon appears next to the address bar; clicking it opens the popup (quick toggles), or you can open the full Options page.

3. Sign in (only required for V2)

Options page → Login tab → "Sign in via web". It opens https://www.live2dweb.com/SingleLogin, and once you complete the login the page closes automatically and writes the access token into chrome.storage.local.

  • The login session is local only; it is never synced across devices.
  • Network requests only happen when you explicitly sign in. The extension never collects URLs / titles of pages you visit.
  • Signing out clears the local token.

If you only want the free V1 models, no sign-in is needed.

4. Model settings

Options page → Basic settings → Model → Model source is a two-way dropdown:

Model sourceMeaningPaid?
Local model (V1)Pick a downloaded model + texture set from the bundled V1 library. First go to the "V1 Library" tab and download the model you want.Free
Custom URL (paid)Plug in your own modelAPI to load V2 (Cubism 4+) models. Same two forms as the WordPress plugin: a direct .json link, or a directory root / + modelDir list.Sign-in + payment required

4.1 V1 model library

Options page → V1 Library tab: lists every available V1 model (with 64×64 thumbnails and status badges). Clicking "Download" stores the model files into the browser's IndexedDB (nothing is uploaded anywhere).

  • Multi-texture type (e.g. 22-musume / 33-musume / Pio / Tia): a single model.moc with multiple texture sets — the Texture ID dropdown is parsed from *.textures.json.
  • Multi-skin type (e.g. Hyperdimension Neptunia / Kantai-Collection / Shizuku Talk): each "skin" is a separate sub-model with its own model.moc; the Texture ID dropdown lists the sub-directory names.

4.2 Custom URL (V2)

After signing in to a paid account, set Model source to "Custom URL (paid)"; modelAPI and modelDir inputs appear. The fill-in rules are identical to the WordPress plugin — see section 3 of the V2 custom model guide.

The browser extension currently has no "cache locally / hotlink protection" switch. The extension fetches the model directly from modelAPI, so make sure CORS allows chrome-extension://... (or *) as an Origin.

5. Other settings tabs

  • Basic settings: enable toggle, model, head anchor (headPos), model scale, character X/Y position, canvas size.
  • Toolbar settings: toggles for the small toolbar buttons (screenshot, switch, settings, …).
  • Tip messages: default tip strings and trigger frequency.
  • Advanced: developer / debug — usually no need to touch.
  • V1 Library: described above.

7. Privacy

The extension does not read URLs, content, cookies, form data or browsing history of pages you visit, and does not collect geolocation / IP / device fingerprints / health info / communications. The full data-usage disclosure is in the browser extension privacy policy.
