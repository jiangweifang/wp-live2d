Live2D Web Canvas · Documentation

This site hosts the documentation for Live2D Web Canvas, which ships as both a WordPress plugin and a Chrome / Edge browser extension.

Notes​

  1. Please respect model copyright. V1 models (journey-ad / Cubism 2~3) have circulated for years and the plugin only ships compatibility loading. For V2 models (Cubism 4+, *.model3.json), keep them on an object storage that supports hotlink protection + CORS, or enable Cache locally in the plugin settings.
  2. Workshop uploads are currently disabled — until 2.0 introduces hotlink-safe storage, we cannot protect uploaders' copyright.
  3. Keep your model textures under 2048 if possible; larger textures noticeably slow loading.
  4. moc3 models often render small by default. Tune the scale toward 2.0. The code intentionally does not hardcode 2.0 because the value compounds (a 1.1 multiplier is actually 2.2).
  5. Make sure your WordPress REST API (rest_route) is reachable. If saving settings fails, check whether the REST API is whitelisted (Cloudflare in particular). Since 2.1.3, the local V1 model service and V2 hotlink protection both rely on /wp-json/live2d/v1/....
  6. CDN users — remember to purge the cache after updating the plugin.
  7. live2d.js (and waifu-tips.min.js etc.) is already minified. Errors after a re-minify pass are usually caused by double-minification.
  8. LAN-only users must access the site via a public IP with a real domain (a free 花生壳 / DDNS domain works). The plugin cannot save settings on plain localhost.
  9. You can buy licensed MOC3 models on nizima.
  10. The official Live2D Showcase features this plugin.
  11. Changelogs are posted on Weibo and bilibili.
  12. If a paid feature does not activate, contact the author at QQ 85838607.
