Things to watch out for

  1. Mind the copyright on your models. V1 models (journey-ad / Cubism 2 ~ 3) have been around for years and the plugin only loads them in a compatibility shim; for V2 models (Cubism 4+, *.model3.json) we recommend hosting them on Aliyun OSS / Tencent COS or similar object storage with referer-based hotlink protection plus CORS, or enabling "cache locally" in the plugin settings to let the plugin guard them for you.
  2. The workshop currently does not accept community uploads — same reason as above. Until the 2.0 anti-leech rewrite ships we cannot protect uploaders' copyright.
  3. Keep texture sizes reasonable; do not exceed 2048, otherwise initial load will be very slow.
  4. moc3 models often need an extra scale factor. Adjust the model scale to a sensible size (around 2.0 tends to work). The code does not hard-code 2.0 because the value is multiplicative (1.1 effectively becomes 2.2).
  5. Make sure your WordPress REST API (rest_route) is reachable. If saving settings fails, check whether your site whitelists the REST API — Cloudflare users hit this most often. Since 2.1.3 the local V1 model service and V2 hotlink protection both rely on /wp-json/live2d/v1/....
  6. If you use a CDN, remember to flush the cache after updating the plugin.
  7. live2d.js (and waifu-tips.min.js etc.) do not need further minification. If errors appear after minifying, undo the second pass first.
  8. If your site is on a private network, please access it via a public IP and configure a domain (a free Peanut Hull domain works). The plugin cannot save settings against a localhost address.
  9. You can purchase legitimate MOC3 models on nizima.
  10. This plugin is featured on the official Live2D Showcase.
  11. Changelogs are posted on Weibo or bilibili.
  12. If a payment does not unlock anything, contact the author on QQ: 85838607; we will sort it out as soon as possible.
