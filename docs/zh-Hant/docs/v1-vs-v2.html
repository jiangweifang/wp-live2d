先看這裡：V1 與 V2 是什麼？

本外掛同時維護兩套各自獨立的 Live2D 載入鏈。第一次接觸本外掛的站長，建議先花 2 分鐘把這頁讀完，再決定要走哪一條路。

一句話分辨

  • V1 模型 = 以 journey-ad/live2d_src 為基礎的 舊版 載入邏輯，對應 Cubism 2.x / 3.x 系列模型，清單檔通常叫 model.json，模型本體副檔名是 .moc。常見模型：22 娘 / 33 娘、Pio / Tia、Shizuku、艦これ Murakumo、超次元海王星等。
  • V2 模型 = Cubism 4 / 5 系列（含未來更新版本），清單檔是 *.model3.json，模型本體副檔名是 .moc3。這是 Live2D 官方目前在維護的格式。

用詞提示：過去的文件曾把 Cubism 4+ 模型稱為「v3 或更高版本」，自外掛 2.1 起一律改稱 V2。也就是說：本倉庫中的 "V2" 不是指 Cubism 2，而是指 本外掛的第二代載入器。若您在 issue 中看到「v3 模型」「V3 模型」字樣，請以 V2 解讀。

怎麼判斷手上的模型是 V1 還是 V2？

看模型清單檔即可：

模型類型清單檔名清單裡的 version 欄位主模型檔
V1（舊）model.json無 / 任意*.moc
V2（新）*.model3.json"Version": 3 起跳（實際是 Cubism Web SDK 版號，並非模型規格號）*.moc3

前端的判斷邏輯是：模型規格版本 ≥ 3 走 V2 路徑，否則走 V1。

WordPress 後台的「創意工坊 API 類型」

後台 → 設定 →「創意工坊 API 類型」是一個三選下拉，對應三種使用方式：

取值意義付費？對應模型
本機 (local)使用外掛 assets/v1/ 內建的 V1 模型，REST 路徑 /wp-json/live2d/v1/model/*，全在自己站台運作，不依賴任何第三方介面。免費V1（22/33/Pio/Tia/Shizuku/HyperdimensionNeptunia/KantaiCollection 等）
遠端創意工坊 (remote)從 www.live2dweb.com 取得模型清單，下載到本機後再走 V1 載入鏈。下載結果放在 wp-content/plugins/live-2d/model/。免費V1（與「本機」同源，只是來源不同）
自訂 (custom)填一個您自己的 modelAPI URL，載入 V2（Cubism 4+）模型。可以指定一個直連 *.model3.json，也可以指定一個目錄根 + 多個 modelDir 子目錄。需付費V2（Cubism 4+ / 5）

為什麼 V2 要付費？

因為 Live2D 株式會社（Live2D Inc.）要求第三方擴充型應用使用 Cubism 4+ 核心時必須簽約。本外掛已與 Live2D 株式會社簽訂長期合約，並依其要求散布正版 Cubism Core，所以 V2 路徑會向站長收取一次性授權費，相當於把上游授權成本分攤給真正使用 V2 的站台。V1 模型不依賴 Cubism Core，因此永遠免費。

我該選哪一個？

  • 只想要會動的二次元小人陪我看部落格 → 安裝外掛、保持預設即可，跑的就是內建的 V1 模型（本機）。
  • 想用市面上現成的小人皮膚（22 娘、Pio 那一系列）→ 也是 V1，可以直接使用「本機」或「遠端創意工坊」。
  • 有自己製作 / 購買的 MOC3 模型 → 走 V2 自訂 URL，先付費，再依照 V2 自訂模型教學設定。
  • 想最大化保護 V2 模型不被盜連 → 在自訂 URL 之外另外開啟「快取至本機」，請參考防盜連頁面。

常見誤解

  • 「V1 是不是不能用了？」 —— 沒有。V1 路徑自 2.1.3 起反而完全本機化（不再依賴 fghrsh.net 等第三方來源），穩定度比過去好得多。它只是不會再加入 新功能（例如混合變形、超採樣這些 Cubism 4+ 才有的特性）。
  • 「V2 = Cubism 2 嗎？」 —— 不是。本外掛的 V2 指的是 第二代載入器，對應 Cubism Web SDK 4 / 5。Cubism 2 是遠古版本，反而歸在 V1 路徑下。
  • 「我看舊文件寫 v3 模型……」 —— 舊文件中的 "v3" 就是現在的 V2。所有「v3 / V3」字樣請一律以 V2 解讀。
