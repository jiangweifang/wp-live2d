V2 模型防盜連：不快取 vs 快取至本機

WordPress 後台 → Live2D 設定 → 基礎設定 → API 方式 決定模型從哪裡載入,以及防盜連由誰阻擋。2026-05 之後舊的 protectV2 二態開關已合併進 apiType,現在是四選項 radio,每一種行為都不同。本頁只談兩種「自訂新版模型」(V2 / Cubism 4+),舊版 V1 模型不走防盜連。

V1（journey-ad / Cubism 2~3）模型不適用本頁。V1 模型本身就由外掛本機 REST 介面（/wp-json/live2d/v1/model/*）發出,不需要再加一層防盜連。本頁所有內容只針對 V2 模型。

四種 API 方式

API 方式 模型放在哪 防盜連由誰擋 什麼時候選
本機部署舊版模型 外掛 model/ 目錄(V1) 不需要 使用 22/33/Pio/Tia 這類 V1 老模型
自行部署舊版模型 站長自己的遠端 V1 PHP API 跨來源,您自己處理(也可以不處理) 已有 fghrsh 風格 V1 介面
自訂新版模型 · 直連載入
舊 protectV2 = direct		 OSS / CDN / GitHub Pages 等跨來源位置 您自己(在物件儲存 / CDN 控制台開 Referer 白名單 / URL 鑑權) 模型已放在阿里雲 OSS、騰訊雲 COS、自家 CDN 等
自訂新版模型 · 託管至本站
舊 protectV2 = local,推薦		 外掛 model/{slug}/ 目錄(V2) 外掛自行處理(自動 alias + 暫時簽名 URL) 模型放在 GitHub Pages / jsDelivr / 自架 nginx 這類無防盜能力的位置,或純本機

選項 A:自訂新版模型 · 直連載入(適合已使用物件儲存 / CDN 的站長)

此模式下外掛不動您的 modelAPI,把它原封不動交給瀏覽器去載入。防盜的事交給物件儲存或 CDN 自行處理。常見三種做法(由簡到繁):

1. 公開讀 + Referer 白名單(最簡單)

  • 阿里雲 OSS:Bucket → 資料安全 → 防盜連,將您的網站網域加入白名單。
  • 騰訊雲 COS:儲存桶 → 安全管理 → 防盜連設定,操作類似。
  • 原理:瀏覽器載入模型時會帶 Referer,不在白名單的來源直接 403。
  • 侷限:稍微懂抓包的人能偽造 Referer 繞過。日常用足夠了。

2. 套一層自家 CDN,開啟 URL 鑑權

在 OSS 前面套阿里雲 CDN / 騰訊雲 CDN / Cloudflare,啟用「URL 鑑權」或「Token 校驗」,再把 modelAPI 填成帶簽名地址。比純 Referer 嚴格,但設定稍複雜。

3. 私有讀 + 簽名連結(最嚴格)

Bucket 設為私有,每次存取都要帶簽名才能下載。此模式無法直接在外掛設定頁設定,需要懂 PHP 的人在佈景主題 functions.php 寫程式碼動態產生簽名 URL。本外掛不內建任何具體雲廠商 SDK,原因見下方 FAQ。

選項 B:自訂新版模型 · 託管至本站(推薦 / 適合無法設定防盜連的來源)

選此項後,模型檔案會被儲存到您的 WordPress 站點(wp-content/plugins/live-2d/model/{slug}/),並對外只暴露一個加密過的暫時簽名 URL:

  • 訪客 F12 看到的也是這個暫時 URL,不會洩漏來源網址。
  • 每個檔案有別名(hash 化路徑),別人猜不到。
  • 2026-05 之後只支援站長手動觸發下載或上傳,不再有訪客存取時的「自動延遲下載」(老站長回饋那種模式不可見、不可控)。
  • 有兩種把模型存到本站的方式,可以共存:

選了「託管至本站」之後表單會怎麼變

選此項後,modelAPI 輸入框會自動隱藏,系統會把它強制寫成本站的 wp-content/plugins/live-2d/model/ 根位址。站長唯一需要填的就是「模型目錄」(modelDir) 列表,每行寫一個 slug(= 子目錄名),外掛會按 {root}{slug}/{slug}.model3.json 的固定範本自動展開為完整的 manifest URL。

例如要載入 haru / Pio / shizuku 三個模型,模型目錄就這樣填:

modelDir:
  - haru
  - Pio
  - shizuku

外掛會自動展開為:

https://your-site.com/wp-content/plugins/live-2d/model/haru/haru.model3.json
https://your-site.com/wp-content/plugins/live-2d/model/Pio/Pio.model3.json
https://your-site.com/wp-content/plugins/live-2d/model/shizuku/shizuku.model3.json

實際訪客拿到的 URL 都是 /wp-json/live2d/v2/m/{token}/{alias}?e=&s= 這樣的暫時簽名連結,來源即使被改成 404 或防盜也不影響。

方式 A:從 URL 下載(模型已經有公開 URL)

  1. 把「API 方式」切到 自訂新版模型 · 託管至本站,儲存。
  2. 頁面下方會出現「模型託管」面板,預設在「從 URL 下載」分頁。
  3. 在「模型來源 URL」輸入框填模型的 .model3.json 完整公開 URL(例如 https://example.com/haru/haru.model3.json),點「下載到本站」。
  4. 外掛會從 URL 派生出 slug(例如 haru),立刻把它追加到上方「模型目錄」列表,同時把整套模型檔案排入佇列,後台 cron 拉取到 model/haru/。
  5. 所有 job 狀態變成「已快取」後,前台就走本機副本了。記得點頁面底部「儲存設定」,modelDir 才會寫入資料庫。

方式 B:上傳資料夾(模型只在本機)

  1. 把「API 方式」切到 自訂新版模型 · 託管至本站,儲存。
  2. 切到「模型託管」面板的「上傳資料夾」分頁,把整個模型目錄(包含 model3.json + moc3 + 貼圖 + 動作)拖到虛線框,或點選「選擇資料夾」按鈕選目錄。
  3. 外掛按相對路徑串列上傳到 model/{slug}/(slug 自動取您拖入的根目錄名),進度條即時反映已上傳位元組。
  4. 上傳完成後 slug 會自動追加到上方「模型目錄」列表。
  5. 頂部黃色橫幅會顯示當前伺服器單檔上傳上限(由 PHP upload_max_filesize 決定)。超過該上限的檔案會被紅色標記跳過,不會讓 PHP 後台靜默丟包。如果模型貼圖普遍偏大,需要聯繫主機商或在 wp-config.php / .htaccess 調高 upload_max_filesize 與 post_max_size。
  6. 上傳完成後記得點頁面底部「儲存設定」,modelDir 才會寫入資料庫。

典型適用場景

  • 模型放在 GitHub Pages / GitHub Raw / jsDelivr。
  • 模型放在朋友的伺服器,沒辦法請對方加 Referer 白名單。
  • 購買了第三方付費模型,對方只給一個公開 URL,您不希望被白嫖。
  • 擔心物件儲存的「按流量計費」帳單失控。
  • 模型只在您本機(沒有任何公開 URL),想直接拖進去就完事。

FAQ:為什麼外掛不直接幫我接 OSS 私有讀?

常有人問「為何不能讓外掛自己拿我的 OSS AccessKey 去簽名轉發?」三個原因:

  1. AccessKey 不能存資料庫。WordPress 的設定項是明文儲存的,萬一資料庫外洩(被駭、備份外流、共用主機出事),您的整個 OSS Bucket 就全沒了,比模型被盜嚴重得多。
  2. 每家雲端的簽名演算法都不一樣。阿里雲、騰訊雲、AWS、Cloudflare R2、MinIO 各搞一套。做完一家就有人提「加 XX 雲支援」,沒完沒了。
  3. 真要用的人不需要外掛幫忙。能搞定 OSS 私有讀的站長,自己寫幾行 PHP 比點設定頁方便得多,也更安全(憑證可走環境變數,不進資料庫)。

想自己改程式碼接 OSS 簽名連結?

目前外掛沒有提供專門的鉤子讓您攔截此流程。如果非改不可,只能 fork 倉庫,去 src/live2d-V2Api.php 改 get_asset() / stream_local() 這兩個函式,讓它在命中您的私有 OSS 路徑時回傳一個 302 跳轉到 OSS 預簽名地址。

如果以後使用此功能的人多了,我們會考慮加一個正式的 live2d_v2_resolve_alias 鉤子,但外掛本身永遠不會內建任何具體雲廠商的 SDK。

如果您真打算這麼做,請務必:

  • 使用最小權限的子帳號(只給「讀取物件」權限,限定到具體 Bucket / 目錄)。
  • 千萬不要把 SecretKey 直接寫進 WordPress 設定項,使用環境變數或專用的密鑰管理服務。
  • 對外掛的 V2 防盜連 REST 路徑加上 Referer 校驗(在 wp-config.php 顯式啟用,參考 live2d-V2Api.php 註解)。
