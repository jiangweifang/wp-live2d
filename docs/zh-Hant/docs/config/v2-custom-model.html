使用 V2 自訂模型（Cubism 4+）

本頁只講 V2（Cubism 4+ / 5）模型。 用 22 / 33 / Pio / Tia 這種舊模型走 V1，不需要付費，請看 V1 模型的兩種使用方式。

1. 先付費​

V2 走的是 Live2D Cubism 4+ Web SDK 載入器。Live2D 株式會社要求第三方擴充性應用使用 Cubism 4+ 時簽訂商業合約，本專案作者已簽長期合約並按其要求分發正版 Cubism Core，所以這條鏈路對站長收一次性授權費。

  1. 在 www.live2dweb.com 完成帳號註冊與付費。
  2. WordPress 後台保持登入狀態（讓外掛能向 live2dweb.com 驗證授權）。

未付費 / 未登入的狀態下，「創意工坊 API 類型」的 自訂 選項會被停用。

2. 準備好 V2 模型檔案​

V2 模型是 Cubism 4+ 匯出的，檔案清單是 *.model3.json，主體是 *.moc3，再加上貼圖、動作、表情等。

檔案清單要求​

  • 清單檔名一般是 {模型名}.model3.json。
  • 路徑裡 不能有非 ASCII / 中文字元。
  • 站點是 https，那麼清單 以及它引用到的所有檔案（.moc3、貼圖、motion3.json、physics3.json、cdi3.json 等）都必須是 https，否則瀏覽器會因混合內容而錯誤。
  • 跨域：把模型放到 OSS / COS / 自家 CDN 時，請正確開啟 CORS（一般加上你的網站網域作為允許的 Origin 即可）。

把模型放在哪裡？（按推薦程度從高到低）​

  1. 物件儲存 + 防盜連（首選，自帶著作權保護）：阿里雲 OSS、騰訊雲 COS、又拍、七牛等。開啟 Referer 白名單，只允許你的網站存取；套一層自家 CDN 時記得繼承防盜連規則。
  2. 「託管到本站」由外掛接管：填 modelAPI 之後，在「API 方式」選「自訂新版模型 · 託管到本站」，外掛會把對應檔案全部下載到 wp-content/plugins/live-2d/model/，並對前端只暴露暫時簽章 URL。詳見 V2 防盜連。
  3. 外掛目錄下的 model 資料夾：放 wp-content/plugins/live-2d/model/{模型名}/{模型名}.model3.json，最簡單但需要你自己控制流量。
  4. GitHub Pages / jsDelivr 這種公開靜態託管：能跑，但 大陸地區不推薦，速度不穩定，也無法防盜連。

3. 在後台填寫 modelAPI​

WordPress 後台 → Live2D 設定 → 創意工坊 API 類型選「自訂」。下面有 modelAPI / modelDir 兩個輸入欄。兩種填法的詳細說明見 modelAPI / modelDir 怎麼填。

4. 模型貼圖（材質）大小​

請控制每張貼圖不超過 2048×2048。Cubism 4+ 預設貼圖就比較大，疊幾張過大的 PNG 很容易把首屏載入時間拖到 5~10 秒以上。如果模型本身匯出時貼圖解析度超標，可以在 Cubism Editor 裡「嵌入紋理圖集 → 重新分配尺寸」。

5. 進階：超取樣（僅 V2）​

外掛 2.1.3 起新增 超取樣 功能，可以讓貼圖在高解析度螢幕上更清晰。僅對 V2（model3.json）模型生效，V1 鏈路不會回應這個開關。

6. 安全 / 著作權要點​

  1. 請自行保證自己模型的著作權。最佳做法：上傳到帶防盜連的物件儲存 + 開啟 CORS 限定來源。
  2. 創意工坊目前不支援自行上傳 —— 因為我們沒辦法替每個上傳者保護著作權。
  3. 不要把 OSS 的 SecretKey 寫進 WordPress 設定項目。如果一定要走私有讀 + 簽章連結，請走 wp-config.php 環境變數，見防盜連頁底部說明。
