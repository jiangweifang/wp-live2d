本插件同时维护着两套完全独立的 Live2D 加载链路。第一次接触本插件的站长，建议先花 2 分钟把这页看完，再决定走哪条路。

一句话区分

  • V1 模型 = 基于 journey-ad/live2d_src 的 老旧 加载逻辑，对应的是 Cubism 2.x / 3.x 系列模型，文件清单一般叫 model.json，模型文件后缀是 .moc。常见模型：22 娘 / 33 娘、Pio / Tia、Shizuku、舰これ Murakumo、超次元海王星等。
  • V2 模型 = Cubism 4 / 5 系列（含未来更高版本），文件清单是 *.model3.json，模型文件后缀是 .moc3。这是 Live2D 官方目前在维护的格式。

术语提示：历史上的文档里曾经把 Cubism 4+ 模型叫"v3 或更高版本"，从插件 2.1 起统一改叫 V2。也就是说：本仓库里的 "V2" 不是指 Cubism 2，而是指 本插件的第二代加载器。如果你在 issues 里看到老的 "v3 模型"、"V3 模型" 字样，请理解为 V2。

怎么判断手里的模型是 V1 还是 V2？

看模型清单文件即可：

模型类型清单文件名清单里的 version 字段主模型文件
V1（旧）model.json无 / 任意*.moc
V2（新）*.model3.json"Version": 3 起步（实际是 Cubism Web SDK 版本号，不是模型规范号）*.moc3

判断逻辑在前端是按 模型规范版本号 ≥ 3 走 V2 链路，否则走 V1 链路。

WordPress 插件后台的"创意工坊 API 类型"

插件后台 → 设置 → 「创意工坊 API 类型」是一个三态下拉，对应三种使用方式：

取值含义付费？对应模型
本地 (local)使用插件 assets/v1/ 内置的 V1 模型，REST 路径 /wp-json/live2d/v1/model/*，完全在你自己的站点提供，不依赖任何第三方接口。免费V1（22/33/Pio/Tia/Shizuku/HyperdimensionNeptunia/KantaiCollection 等）
远程创意工坊 (remote)从 www.live2dweb.com 拉模型清单，下载到本地后再走 V1 加载链路。下载结果落到 wp-content/plugins/live-2d/model/。免费V1（与"本地"同源，只是来源不同）
自定义 (custom)填一个你自己的 modelAPI URL，加载 V2（Cubism 4+）模型。可以指定一个直链 *.model3.json，也可以指定一个目录根 + 多个 modelDir 子目录。需要付费V2（Cubism 4+ / 5）

为什么 V2 要付费？

因为 Live2D 株式会社（Live2D Inc.）要求第三方扩展性应用使用 Cubism 4+ 核心时必须签约。本插件已经和 Live2D 株式会社签订了长期合约并按其要求分发正版 Cubism Core，所以 V2 链路对站长这边收取一次性授权费，相当于把上游许可成本分摊给真正使用 V2 的站点。V1 模型不依赖 Cubism Core，因此始终免费。

我应该选哪个？

  • 只想要会动的二次元小人陪着看博客 → 装上插件、保持默认设置即可，跑的就是内置 V1 模型（本地）。
  • 想用市面上各种现成的小人皮肤（22 娘、Pio 那一系列）→ 也是 V1，可以直接用"本地"或"远程创意工坊"。
  • 有自己制作 / 购买的 MOC3 模型 → 走 V2 自定义 URL，先付费、再按 V2 自定义模型教程 配置。
  • 想最大化保护 V2 模型不被盗链 → 在自定义 URL 之上额外开启「缓存到本地」，参考防盗链页。

常见误解澄清

  • "V1 是不是不能用了？" —— 没有。V1 链路从 2.1.3 起反而完全本地化（再不用 fghrsh.net 等第三方源），稳定性比以前好得多。它只是不会再加 新功能（比如融合变形、超采样这些 Cubism 4+ 才有的特性）。
  • "V2 = Cubism 2 吗？" —— 不是。本插件里的 V2 指的是 第二代加载器，对应 Cubism Web SDK 4 / 5。Cubism 2 是远古版本，反而归在 V1 链路里。
  • "我看老文档说 v3 模型……" —— 老文档里的 "v3" 就是现在的 V2。所有"v3 / V3"字样统一按 V2 理解。
