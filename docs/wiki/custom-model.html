如何使用 v3 或更高版本的自定义模型

← 返回首页 ChatGPT 的使用 触摸区域 原始 Wiki

  • 感谢您的使用。在这段时间里，看到很多用户问我如何使用 Live2D Cubism v3 或以上版本（现在已经是 4.x 快到 5）的朋友问我如何使用他们自己制作的模型。
  • 本 Wiki 是给您演示如何使用这种模型。

1. 首先：请付费

由于 Live2D 株式会社要求在使用其扩展时必须付费，本人（个人开发者）和 Live2D 株式会社签订了长期合约，按照其要求提供正版的插件。

关于 ChatGPT，可以参考此 Wiki

付费后，请确保完整地登录了 WordPress 后台，以便 Token 的正常通讯。

2. 请正确地上传您自己制作的 Live2D 模型

通过 B 站各位 UP 主的模型制作教学，您可以得知以下信息。

2.1 文件清单（*.model3.json）

您需要确保上传 Live2D 模型后可以正确地访问文件清单。

  • 文件清单的路径不可以为中文；
  • 如果您的站点处于 https 状态，您需要保证文件清单及其全部文件都保持在 https 状态下，否则会出现跨域异常。

您可以将做好的模型文件夹上传到您站点的目录，使用 Nginx 反向代理来给它指定一个二级域名，也可以上传到 WordPress 的路径中；但是由于 WordPress 的自我保护（URL Rewrite），您需要找到一个更准确的上传地址才可以直接访问到此模型及其清单文件。

如果您没有办法找到正确的存放位置，您可以保存到插件目录下的 model 文件夹中。如果没有这个文件夹，可以创建一个，它看起来像这样：

保存好之后，您就可以获得这个路径，例如：

https://[您的域名]/wp-content/plugins/live-2d/model/[模型名称]/[模型名称].model3.json

【推荐方法】 您也可以将模型上传至阿里云 OSS 或者腾讯云 COS，以保证版权不被侵害。

当然，您可能有更好的办法，比如放到 GitHub 中，例如：

  • https://raw.githubusercontent.com/Live2D/CubismWebSamples/develop/Samples/Resources/Mao/Mao.model3.json
  • https://cdn.jsdelivr.net/gh/live2d/CubismWebSamples@develop/Samples/Resources/Mao/Mao.model3.json

但是即使您使用 jsdelivr，仍然会被指向原始地址，所以在大陆地区不建议这样使用。

2.2 请将您得到的路径复制到后台 API 部分

使用您自己的模型将会导致前台无法切换模型，请在后台设置关闭那两个按钮。

2.3 模型皮肤（材质）

您需要控制您的模型材质，不要过大，建议到 2048 即可，否则加载模型的时间会很长。

3. 安全问题

  1. 请自行保证自己模型的版权。您可以通过上传到阿里云 OSS 或者腾讯云 COS，通过防盗链和跨域保护来保证自己的模型不被盗用。
  2. 创意工坊目前不支持自行上传，因为我不能通过开源代码保护您的版权。
