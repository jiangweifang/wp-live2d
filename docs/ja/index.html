利用ガイド

まずはここから:V1 と V2 とは?

本プラグインには 2 系統の Live2D ロード経路があります。V1 = journey-ad ベースの旧ローダー(Cubism 2 / 3)、V2 = Cubism 4+(5 を含む)。見分け方と用途を解説します。

V2 カスタムモデル(Cubism 4+)を使う

有償ライセンス、モデルの設置場所、modelAPI の 2 通りの書き方、modelDir による複数モデル切り替え、テクスチャサイズの目安。

V2 モデルの直リンク防止: パススルー vs ローカルキャッシュ

2 つの排他オプションの選び方、なぜプラグインは OSS の Private 読み出しを直接代行しないのか、すでにオブジェクトストレージや CDN を使っているサイト向けの補足。

Live2D Web Canvas ブラウザ拡張

Chrome / Edge 拡張のインストール、ログイン、組み込みの V1 モデルライブラリ、有償のカスタム URL モード、WordPress プラグインとの関係。

ChatGPT の使い方

トークン数の設定、Open AI / Azure OpenAI エンドポイントの差し替え。既定値は MaxTokens=200、コンテキストは直近 5 ターンまで。

V2 モデルの当たり判定 (HitArea)

model3.json の HitAreas とモーショングループの対応関係(簡易リファレンス)。

ブラウザ拡張プライバシーポリシー

Chrome / Edge 拡張 Live2D Web Canvas のプライバシーポリシー。

V1 モデルサーバを自前で立てる(外部)

fghrsh がメンテナンスしている旧来の Live2D API(V1)。バージョン 2.1.3 以降は本プラグインが主要な V1 モデルを同梱しているため、通常はサードパーティ API は不要です。

注意事項

  1. モデルの著作権は各自で保護してください。V1 モデル(journey-ad / Cubism 2 ~ 3 系)は長年流通してきたものを互換ローダーで読み込むだけです。V2 モデル(Cubism 4+、*.model3.json)は Aliyun OSS / Tencent COS などの直リンク防止 + CORS が利くオブジェクトストレージに置くか、プラグイン設定で「ローカルキャッシュ」を有効にして保護してください。
  2. クリエイティブワークショップは現在アップロードを受け付けていません。理由は同じ ── 2.0 のアンチリーチ改修が入るまで、投稿者の著作権を守れないためです。
  3. テクスチャサイズはほどほどに。2048 を超えないようにしてください。初回ロードが非常に遅くなります。
  4. moc3 モデルは出力時のスケールが小さい場合があります。表示倍率を適切な値(目安は 2.0 前後)に調整してください。コードに 2.0 を直書きしていないのは、設定値が乗算で効く(1.1 が実質 2.2 になる)ためです。
  5. WordPress の REST API(rest_route)が外部から叩けることを必ず確認してください。設定保存に失敗する場合、サイトが REST API をホワイトリストから外していないか確認してください ── 特に Cloudflare 利用時に発生しやすいです。2.1.3 以降、ローカル V1 モデル配信と V2 直リンク防止はすべて /wp-json/live2d/v1/... 経由です。
  6. CDN を使っている方は、プラグイン更新後に必ずキャッシュを破棄してください。
  7. live2d.js(および waifu-tips.min.js など)は追加圧縮不要です。圧縮後にエラーが出た場合は、まず二重圧縮を疑ってください。
  8. イントラネット環境のユーザーは、必ず公開 IP と独自ドメイン(花生殻の無料ドメインなどで可)でアクセスしてください。localhost ではプラグインの設定が保存できません。
  9. 正規の MOC3 モデルは nizima で購入できます。
  10. 本プラグインは公式 Live2D Showcase に掲載されています。
  11. 更新情報は Weibo または bilibili で告知します。
  12. 決済後に反映されない場合は、作者の QQ: 85838607 までご連絡ください。可能な限り速やかに対応します。
