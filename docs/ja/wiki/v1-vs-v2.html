本プラグインは 2 系統の完全に独立した Live2D ローダーを並行してメンテナンスしています。初めてこのプラグインを触る方は、まず 2 分ほどでこのページに目を通してから、どちらを使うか決めてください。

一行サマリ

  • V1 モデル = journey-ad/live2d_src ベースの旧来ローダー。Cubism 2.x / 3.x 系のモデルが対象で、マニフェストは通常 model.json、モデル本体の拡張子は .moc です。代表例:22 娘 / 33 娘、Pio / Tia、Shizuku、艦これ Murakumo、超次元ネプテューヌなど。
  • V2 モデル = Cubism 4 / 5 系(以降のバージョンも含む)。マニフェストは *.model3.json、モデル本体は .moc3。Live2D Inc. が現在メンテナンスしているフォーマットです。

用語の補足:古いドキュメントでは Cubism 4+ モデルを「v3 以上」と呼んでいましたが、プラグイン 2.1 以降は V2 に統一しました。本リポジトリの "V2" は Cubism 2 のことではなく、本プラグインの第 2 世代ローダーを指します。古い issue で「v3 モデル」「V3 モデル」と書かれていたら、すべて V2 と読み替えてください。

手元のモデルが V1 か V2 かを判別する方法

マニフェストファイルを見るだけです:

モデル種別マニフェスト名マニフェスト内の version本体ファイル
V1(旧)model.jsonなし / 任意*.moc
V2(現行)*.model3.json"Version": 3 以上(これは Cubism Web SDK のバージョンであり、モデル仕様番号ではありません)*.moc3

フロントエンド側のロジックは、モデル仕様バージョンが 3 以上なら V2、それ以下なら V1 にフォールバックします。

WordPress 管理画面の「クリエイティブワークショップ API タイプ」

管理画面 → 設定 →「クリエイティブワークショップ API タイプ」は 3 値のドロップダウンです:

値意味有償?モデル種別
ローカル(local)プラグイン同梱の assets/v1/ 内 V1 モデルを、REST ルート /wp-json/live2d/v1/model/* 経由で配信します。完全に自サイト内で動作し、サードパーティ API は一切不要です。無料V1(22/33/Pio/Tia/Shizuku/HyperdimensionNeptunia/KantaiCollection など)
リモートワークショップ(remote)www.live2dweb.com からモデルカタログを取得し、ローカルにダウンロードしてから V1 ローダーに通します。ダウンロード先は wp-content/plugins/live-2d/model/。無料V1(中身は「ローカル」と同じ、配信元だけが違います)
カスタム(custom)自分の modelAPI URL を入力して V2(Cubism 4+)モデルをロードします。単一の *.model3.json 直リンクも、ディレクトリルート + 複数の modelDir 指定も可能。有償V2(Cubism 4+ / 5)

なぜ V2 は有償なのか?

Live2D 株式会社が、サードパーティ製の拡張可能アプリケーションが Cubism 4+ Core を利用する際に商用契約を要求しているためです。本プラグインの作者はすでに長期契約を締結し、その契約のもとで正規の Cubism Core を再配布しています。そこで V2 経路ではサイトオーナーから一括のライセンス料を頂戴し、上流のライセンスコストを実際に V2 を使うサイトに振り分けています。V1 モデルは Cubism Core に依存しないため、永久に無料です。

どちらを選べばよいか

  • ブログのおまけに動く 2D キャラを乗せたいだけ → 既定のままインストールすれば、すでに同梱された V1 モデル(ローカル)が動きます。
  • 市販の有名スキン(22 娘、Pio など)を使いたい → こちらも V1 です。「ローカル」または「リモートワークショップ」をそのまま使ってください。
  • 自作 / 購入した MOC3 モデルがある → V2 のカスタム URL を使います。先に課金を済ませ、V2 カスタムモデルガイドに従って設定してください。
  • V2 モデルの直リンク防止を最優先したい → カスタム URL の上にさらに「ローカルキャッシュ」を有効化してください。詳細は直リンク防止ページへ。

よくある誤解

  • 「V1 はもう使えないのでは?」 いいえ。2.1.3 以降は V1 ローダーが完全ローカル化(fghrsh.net 等のサードパーティに依存しなくなった)されたため、以前より格段に安定しています。ただ Cubism 4+ 専用の新機能(ブレンドシェイプやスーパーサンプリング等)は今後も追加されません。
  • 「V2 = Cubism 2 ですか?」 違います。本プラグインの V2 は第 2 世代ローダーを意味し、Cubism Web SDK 4 / 5 に対応します。Cubism 2 は太古のバージョンで、こちらは V1 ローダー側に属します。
  • 「古いドキュメントに v3 モデルと書いてあるが…」 古いドキュメントの "v3" は今の V2 のことです。"v3 / V3" の表記はすべて V2 と読み替えてください。
