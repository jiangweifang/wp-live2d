このページは V2(Cubism 4+ / 5)モデル専用です。22 / 33 / Pio / Tia のような旧モデルを使いたい場合は V1 ですので、プラグイン管理画面で「クリエイティブワークショップ API タイプ = ローカル / リモートワークショップ」を選択してください。課金は不要で、このページのどの手順も該当しません。先に V1 / V2 とは をご覧ください。

1. まず課金する

V2 モデルは Live2D Cubism 4+ Web SDK ローダーを通ります。Live2D 株式会社は、サードパーティ製の拡張可能アプリケーションが Cubism 4+ を使用する際に商用契約を要求しているため、本プラグイン作者はすでに長期契約を締結し、その契約のもとで正規の Cubism Core を再配布しています。そこでこの経路ではサイトオーナーから一括のライセンス料を頂戴します。

  1. www.live2dweb.com でアカウント登録と決済を完了します。
  2. WordPress 管理画面にログインしたままにしてください(アカウントに紐付いたトークンを取得し、プラグインから live2dweb.com にライセンス検証できるようにするためです)。

未課金 / 未ログイン状態では、「クリエイティブワークショップ API タイプ」の カスタム 項目は無効化されるか、ログインを促されます。

2. V2 モデルファイルを用意する

V2 モデルは Cubism 4+ から書き出したもので、マニフェストは *.model3.json、本体は *.moc3、加えてテクスチャ・モーション・表情などです。

2.1 マニフェストの要件

  • マニフェスト名は通常 {model-name}.model3.json。
  • パスに 中国語の文字を含めないでください。
  • サイトが https なら、マニフェストとそこから参照される全てのファイル(.moc3、テクスチャ、motion3.json、physics3.json、cdi3.json など)も https である必要があります。さもないとブラウザが Mixed Content / CORS で拒否します。
  • CORS:OSS / COS / 自社 CDN に置く場合は CORS を正しく設定してください(自サイトのドメインを Origin として許可するだけで十分なケースが多いです)。

2.2 モデルはどこに置く?

推奨度の高い順:

  1. オブジェクトストレージ + 直リンク防止(おすすめ・著作権保護込み):Aliyun OSS、Tencent COS、Upyun、Qiniu など。
    • Referer ホワイトリストを有効化し、自サイトのみ許可。
    • 前段に CDN を挟む場合は、CDN 側にも同等の直リンク防止ルールを継承させてください。
  2. プラグイン配下の model ディレクトリ(最も簡単。ただし帯域は自分で管理が必要): 
    wp-content/plugins/live-2d/model/{model-name}/{model-name}.model3.json
    model フォルダが無ければ自分で作成してください。テーマやセキュリティプラグインによっては WordPress の URL Rewrite が誤検知でブロックすることがあります。読み込めない場合は、まずブラウザでその URL を直接開けるか確認してください。
  3. 「ローカルキャッシュ」を有効にしてプラグインに任せる:modelAPI を入力した上で、直リンク防止オプションを「ローカルキャッシュ」に切り替えると、プラグインが対象ファイル一式を wp-content/plugins/live-2d/model/ に取り込み、フロントには一時的な署名 URL のみを露出します。F12 でも元 URL は見えません。直リンク防止ページを参照。
  4. GitHub Pages / jsDelivr のような公開静的ホスティング:動作はしますが 中国本土での運用は非推奨です。jsDelivr は最終的に raw.githubusercontent.com にフォールバックするため速度が安定せず、直リンク防止もできません。

3. 管理画面で modelAPI を入力する

WordPress 管理画面 → Live2D 設定 → クリエイティブワークショップ API タイプ:「カスタム」を選択。下に modelAPI 入力欄があります。2 通りの書き方があり、末尾文字でどちらかが決まります:

書き方末尾動作下の modelDir 一覧
① 単一モデル直リンク.json で終わるその model3.json をそのままロード。パスの追加結合は行いません。無視
② ディレクトリルート + 複数モデル/ で終わるルートディレクトリとみなし、下の modelDir 一覧と組み合わせて {modelAPI}{name}/{name}.model3.json を自動生成します。必須。1 行 1 サブディレクトリ名({name}.model3.json と完全一致が必要)。

例

① 直リンク:

https://cdn.example.com/live2d/haru/haru.model3.json

② ディレクトリルート + 複数モデル:

modelAPI:  https://cdn.example.com/live2d/

modelDir:
  - haru
  - mao
  - shizuku

ロード時に以下の URL に展開されます:

https://cdn.example.com/live2d/haru/haru.model3.json
https://cdn.example.com/live2d/mao/mao.model3.json
https://cdn.example.com/live2d/shizuku/shizuku.model3.json

フロントエンドの「まばたきボタン」がこれらのモデル間を巡回します。

注意:カスタムモデルを利用する場合、フロントエンドの「前のキャラ / 次のキャラ」ボタンは多くの組み合わせで意味を持ちません。管理画面で OFF にすることをおすすめします。

4. テクスチャ(マテリアル)サイズ

1 枚あたり 2048×2048 を超えないように。Cubism 4+ はそもそもテクスチャが大きいため、過大な PNG を何枚も重ねると初回描画が 5~10 秒以上かかってしまいます。書き出し時にサイズが大きすぎる場合は、Cubism Editor の「テクスチャアトラス埋め込み → サイズ再割り当て」で再パックできます。

5. 上級設定:スーパーサンプリング(V2 限定)

プラグイン 2.1.3 で スーパーサンプリング機能を追加しました。高 DPI ディスプレイでテクスチャがよりシャープになります。V2(model3.json)モデルにのみ有効で、V1 経路はこのスイッチに反応しません。

6. セキュリティ / 著作権の要点

  1. モデルの著作権は各自で保証してください。ベストプラクティス:Referer 直リンク防止つきオブジェクトストレージにアップロードし、CORS を自サイトのみに限定。
  2. クリエイティブワークショップは現在、自由なアップロードに対応していません。投稿者ごとの著作権を守る術がまだ無いためです。
  3. OSS の SecretKey を WordPress の設定項目に書き込まないでください。Private 読み出し + 署名 URL を運用する場合は、wp-config.php の環境変数経由にしてください。直リンク防止ページの末尾を参照。
