このページは V2(Cubism 4+ / 5) モデル、すなわち *.model3.json 専用です。V1(journey-ad / Cubism 2~3)モデルの当たり判定は別の仕組み(hit_areas_custom ベース)で扱われ、本ページの対象外です。

1. 当たり判定はどこに書く?

V2 モデルの当たり判定は *.model3.json のトップレベル HitAreas フィールドで定義します。各エントリには Name と対応する ArtMesh ID があり、ブラウザは Cubism Core で ArtMesh の当たり判定ボックスを計算し、ボックス内のクリックを当該 HitArea のヒットとみなします。

{
  "Version": 3,
  "FileReferences": { ... },
  "HitAreas": [
    { "Name": "Head",   "Id": "HitAreaHead"   },
    { "Name": "Body",   "Id": "HitAreaBody"   }
  ]
}
  • Name:論理名。プラグインはこの名前を見てモーションを振り分けます。慣例として Head と Body を使い、それぞれ「頭」「体」に対応します。
  • Id:Cubism Editor でその ArtMesh に付けた ID。Editor 上でメッシュを選択 → 右側のインスペクタで参照 / 変更できます。

2. HitArea とモーショングループの対応関係

本プラグインの振り分けルールは Live2D 公式 CubismWebSamples と一致しています:

タップした HitArea発火するモーショングループmodel3.json の Motions に必要なグループ
Head表情切替(ランダムに 1 つ)Expressions フィールド(Motions ではない)
Body体タップモーショングループMotions.TapBody

model3.json に Expressions / TapBody が無い場合、その領域をタップしても何も起こりませんが、エラーにもなりません。

3. Cubism Editor で ArtMesh を HitArea として登録するには?

  1. .cmo3 プロジェクトを開き、モデリングビューに切り替えます。
  2. 当たり判定にしたい ArtMesh(通常は頭部 / 体を覆っているメッシュ)を選択。
  3. 右側インスペクタ → ID。識別しやすい ID(例:HitAreaHead)を控える/変更します。
  4. .moc3 を書き出す際に「テクスチャアトラスを埋め込む」をチェックし、.model3.json を生成。
  5. テキストエディタで .model3.json を開き、HitAreas 配列に { "Name": "Head", "Id": "HitAreaHead" } を追加します(書き出し時に「HitArea を含む」をチェックしていれば Editor が自動で書きます)。

4. デバッグのコツ

  • タップしても反応しない場合は、まずブラウザコンソールに motion / expression のロード失敗エラーが無いか確認してください。
  • 当たり判定は ArtMesh の実際の可視メッシュに基づきます。ArtMesh の不透明度を 0 にして隠していても、当たり判定ボックスは生きています。「何も見えていないのにクリック判定がある」という現象の典型的な原因です。
  • モデル作者によって命名が一致しません。ある人は HitAreaHead、ある人は head、極端な例では全部 HitArea という名前のことすらあります。本プラグインは Name で振り分けるので、必ず Name を Head または Body にしてください。
