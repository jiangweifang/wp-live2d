WordPress プラグイン以外に、本プロジェクトでは Chrome / Edge ブラウザ拡張 Live2D Web Canvas も提供しており、任意のウェブページの右下隅に Live2D キャラクターを表示できます。WordPress プラグインとは独立した 2 つの製品で、ログインアカウントは共通ですが設定は完全に別管理です。

1. 何ができるのか?

  • あなたが訪れる任意のサイトの右下隅に Live2D キャラを表示します(ホストページの内容は一切読まず、マウス位置のみで向きアニメーションを行います ── プライバシーポリシー参照)。
  • 定番の V1 モデルを同梱(22 娘 / 33 娘、Pio / Tia、Shizuku、艦これ Murakumo、超次元ネプテューヌなど)、IndexedDB に保存してオフライン利用可。
  • 有償アカウントにログインすれば、自分の modelAPI を入力して V2(Cubism 4+)モデルを読み込めます。
  • 頭部アンカー、表示倍率、キャラ位置、キャンバスサイズ、提示バブルの頻度、ツールバーボタン等を調整可。

2. インストール

Chrome Web Store / Edge アドオンストアで Live2D Web Canvas を検索してインストールしてください。インストール後、アドレスバー右側に拡張アイコンが現れ、クリックでポップアップ(クイックトグル)を表示、または「オプション」ページ(完全な設定)を開けます。

3. ログイン(V2 を使う場合のみ必要)

オプションページ → ログイン タブ →「Web ログインを使う」。https://www.live2dweb.com/SingleLogin へ遷移し、ログインが完了するとページが自動的に閉じ、アクセストークンが chrome.storage.local に書き込まれます。

  • ログイン状態はローカル限定で、デバイス間同期はされません。
  • ネットワークアクセスはユーザーが明示的にログインした場合のみ発生します。拡張は閲覧 URL / タイトルを収集しません。
  • ログアウトすればローカルトークンは削除されます。

無料の V1 モデルだけ使うならログインは不要です。

4. モデル設定

オプションページ → 基本設定 → モデル → モデルソース は 2 値のドロップダウンです:

モデルソース意味有償?
ローカルモデル(V1)同梱 V1 モデルライブラリからダウンロード済みのモデル + テクスチャを選択。先に「V1 ライブラリ」タブで使いたいモデルをダウンロードしてください。無料
カスタム URL(有償)自分の modelAPI を入力して V2(Cubism 4+)モデルをロード。記法は WordPress プラグインと同じ:.json 直リンク、または / で終わるディレクトリルート + modelDir 一覧。ログイン + 有償

4.1 V1 モデルライブラリ

オプションページ → V1 ライブラリ タブ:利用可能な V1 モデル一覧(64×64 サムネイル + ステータスバッジ)が表示されます。「ダウンロード」を押すとモデルファイルがブラウザの IndexedDB に保存されます(どこにもアップロードされません)。

  • マルチテクスチャ型(22 娘 / 33 娘 / Pio / Tia など):単一の model.moc に複数のテクスチャセット。Texture ID ドロップダウンは *.textures.json から解析されたもの。
  • マルチスキン型(超次元ネプテューヌ / 艦これ / Shizuku Talk など):各「スキン」が独立サブモデル(各々の model.moc)、Texture ID ドロップダウンにはサブディレクトリ名が並びます。

4.2 カスタム URL(V2)

有償アカウントでログイン後、モデルソースを「カスタム URL(有償)」にすると、modelAPI と modelDir の入力欄が現れます。記法は WordPress プラグインと完全に同じ ── V2 カスタムモデルガイド第 3 節を参照してください。

ブラウザ拡張側には現在「ローカルキャッシュ / 直リンク防止」スイッチはありません。拡張は modelAPI から直接モデルを取得するため、modelAPI 自体が CORS に対応している必要があります(chrome-extension://... または * を Origin として許可してください)。

5. その他の設定タブ

  • 基本設定:有効/無効、モデル、頭部アンカー(headPos)、モデル倍率、キャラ位置(X/Y)、キャンバスサイズ。
  • ツールバー設定:右下のツールボタン(スクショ・切替・設定など)のオン/オフ。
  • 提示メッセージ:既定の提示文言、トリガー頻度。
  • 高度な設定:開発・デバッグ用。通常は触らなくて構いません。
  • V1 ライブラリ:上記参照。

6. WordPress プラグインとの関係

WordPress プラグインブラウザ拡張
動作場所自分の WP サイトユーザーのブラウザ、すべてのウェブページ
誰が設定するかサイトオーナー(一度の設定で全サイトに反映)各ユーザーが各々のブラウザで設定
V1 モデルソース同梱 + リモートワークショップ同梱(IndexedDB キャッシュ)
V2 カスタム URL有償 + 「ローカルキャッシュ」オプション有償、ローカルキャッシュなし(直接取得)
アカウント同じ live2dweb.com アカウントですが、アクセストークンは共有されないため別途ログインが必要

7. プライバシー

拡張は閲覧 URL、ページ内容、Cookie、フォーム入力、ブラウジング履歴を読み取りません。位置情報 / IP / デバイス指紋 / 健康情報 / 通信内容も収集しません。完全なデータ使用一覧は ブラウザ拡張プライバシーポリシー をご覧ください。
